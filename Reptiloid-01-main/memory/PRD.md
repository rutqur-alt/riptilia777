# Reptiloid P2P Exchange - PRD

## Описание проекта
P2P криптовалютная биржа с интегрированным маркетплейсом и платёжным шлюзом для мерчантов.

## Основной функционал

### 1. Платёжный шлюз для мерчантов
- **Путь:** `/select-operator/:invoiceId`
- **Demo магазин:** `/demo`
- **Логика:** Клиент мерчанта видит только "пополнение счёта", без крипто-терминологии
- **Фильтры:** По способу оплаты (карта, СБП, SIM)
- **Комиссия оператора:** Отображается в процентах вместо "курса"

### 2. Процесс оплаты
1. Выбор оператора (как в стакане)
2. Выбор реквизита (если несколько)
3. Принятие правил
4. Оплата с таймером 30 минут
5. Чат с оператором
6. Спор через 10 минут после "Я оплатил"

### 3. История чатов (для админов/модераторов)
- **P2P сделки:** `/admin/p2p/trades` → кнопка "История чата"
- **Маркетплейс:** `/admin/market/orders` → кнопка "История чата"
- Сохраняются ВСЕ сообщения и реквизиты
- Работает для всех статусов (завершённые, отменённые, споры)

### 4. Реквизиты в чате
При создании сделки реквизиты ВСЕГДА записываются в чат:
- 💳 Банковская карта (номер + получатель)
- ⚡ СБП (телефон)
- 📱 Мобильный (телефон)
- 📲 QR-код

## Роли пользователей
- **admin** - полный доступ
- **mod_p2p** - модерация P2P сделок
- **mod_market** - модерация маркетплейса
- **trader** - трейдер/оператор
- **merchant** - мерчант с API

## API Endpoints

### Публичные
- `GET /api/public/operators` - список операторов для оплаты
- `GET /api/shop/pay/:invoiceId` - данные платежа
- `POST /api/shop/quick-payment` - создание тестового платежа
- `POST /api/trades` - создание сделки
- `GET /api/trades/:id/public` - данные сделки (публичные)
- `POST /api/trades/:id/mark-paid` - отметить как оплачено
- `POST /api/trades/:id/dispute-public` - открыть спор

### Админские
- `GET /api/admin/trades-history` - история P2P сделок
- `GET /api/admin/trades-history/:id` - детали сделки
- `GET /api/admin/marketplace-history` - история заказов маркетплейса
- `GET /api/admin/marketplace-history/:id` - детали заказа
